## <a name="ways-in-which-the-book-is-outdated"></a>Sposoby, w którym książki jest nieaktualna

Od momentu opublikowania <i>tworzenia aplikacji mobilnych za pomocą zestawu narzędzi Xamarin.Forms</i>, dodano kilka nowych funkcji do zestawu narzędzi Xamarin.Forms. Te nowe funkcje są opisane w poszczególnych artykuły w [Xamarin.Forms](/xamarin/) dokumentacji. 

Inne zmiany spowodowały część zawartości książki jako przestarzałe:

### <a name="net-standard-20-libraries-have-replaced-portable-class-libraries"></a>Biblioteki .NET standard 2.0 zastąpiono biblioteki klas przenośnych

Aplikacja Xamarin.Forms ogólnie używa biblioteki do udostępniania kodu między różnymi platformami. Początkowo jest to biblioteka klas przenośnych (PCL). Istnieje wiele odwołań do PCLs książki i podsumowanie rozdziałów. 

Portable Class Library, jest zastępowany przy użyciu biblioteki .NET Standard 2.0, zgodnie z opisem w artykule [.NET Standard 2.0 pomocy technicznej w interfejsie Xamarin.Forms](~/xamarin-forms/internals/net-standard.md). Wszystkie [przykładowego kodu](https://github.com/xamarin/xamarin-forms-book-samples) z książki został zaktualizowany do użycia biblioteki .NET Standard 2.0.

Większość informacji w książce dotyczących roli Portable Class Library pozostaje taki sam, biblioteki .NET Standard 2.0. Jedną różnicaą jest to, że tylko aplikacji PCL ma numeric "profil". Są tam również niektóre zalety do bibliotek .NET Standard 2.0. Na przykład działu 20 [Async i we/wy](summaries/chapter20.md) w tym artykule opisano sposób użycia platform podstawowych do wykonywania operacji We/Wy w pliku. To nie jest już konieczne. Biblioteka .NET Standard 2.0 obsługuje znanej [System.IO](xref:System.IO) klasy dla wszystkich platform zestawu narzędzi Xamarin.Forms.

Biblioteki .NET Standard 2.0 umożliwia aplikacjom używanie zestawu narzędzi Xamarin.Forms [ `HttpClient` ](xref:System.Net.Http.HttpClient) uzyskiwania dostępu do plików za pośrednictwem Internetu zamiast [ `WebRequest` ](xref:System.Net.WebRequest) lub innych klas.

### <a name="the-role-of-xaml-has-been-elevated"></a>Podwyższony poziom roli XAML

<i>Tworzenie aplikacji mobilnych za pomocą zestawu narzędzi Xamarin.Forms</i> rozpoczyna się od opisujące sposób pisania aplikacji platformy Xamarin.Forms przy użyciu języka C#. Extensible Application Markup Language (XAML) nie są wprowadzane do [rozdział 7. XAML vs. Kod](summaries/chapter07.md). 

XAML ma teraz znacznie ważniejszą rolę w interfejsie Xamarin.Forms. Szablony rozwiązań platformy Xamarin.Forms, rozpowszechniać za pomocą programu Visual Studio Utwórz pliki stronicowania opartych na XAML. Developer, przy użyciu zestawu narzędzi Xamarin.Forms należy zapoznać się z XAML możliwie jak najszybciej. [EXtensible Application Markup Language (XAML)](~/xamarin-forms/xaml/index.md) sekcji dokumentacji zestawu narzędzi Xamarin.Forms zawiera kilka artykułów na temat XAML, które ułatwią Ci rozpoczęcie pracy.

### <a name="supported-platforms"></a>Obsługiwane platformy 

Zestaw narzędzi Xamarin.Forms nie obsługuje już Windows 8.1 i Windows Phone 8.1. 

Książki czasami powoduje odwołania do _Windows Runtime_. Jest to okres, który obejmuje interfejs API Windows, używany w wielu wersjach systemu Windows i Windows Phone. Nowsze wersje zestawu narzędzi Xamarin.Forms ogranicza się do obsługi platformy Windows uniwersalnej, czyli interfejsu API dla systemu Windows 10 i Windows 10 Mobile.

Biblioteka .NET Standard 2.0 nie obsługuje dowolnej wersji systemu Windows 10 Mobile. W związku z tym aplikację platformy Xamarin.Forms przy użyciu biblioteki .NET Standard nie uruchomi się na urządzeniu z systemem Windows 10 Mobile. Aplikacje Xamarin.Forms w dalszym ciągu uruchamiać na pulpicie systemu Windows 10 w wersji 10.0.16299.0 lub nowszym. 

Zestaw narzędzi Xamarin.Forms obejmuje obsługę wersji zapoznawczej [Mac](~/xamarin-forms/platform/mac.md), [WPF](~/xamarin-forms/platform/wpf.md), [GTK #](~/xamarin-forms/platform/gtk.md), i [Tizen](/xamarin-forms/platform/tizen.md) platform.
